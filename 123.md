🧠 七、给你一个“成熟架构建议”（很实用）
🎯 推荐标准模型
LF-Tag:
  domain        → 数据归属
  sensitivity   → 数据敏感度
🎯 用户 Tag 来源
domain        ← AD group
sensitivity   ← 权限系统（不是 AD）

👉 ❗重点：

高敏权限不要直接挂 AD group
🚀 最后给你一个判断标准（很好用）

你设计 tag 时问自己：

这个 tag 是“数据属性”还是“用户角色”？
类型	是否适合 LF-Tag
数据属性（sensitivity）	✅
用户角色（admin）	❌

高敏列必须通过 LF-Tag 与用户 Session Tag 匹配来控制；
sensitivity level 完全可自定义，但不具备层级关系，需要通过设计来实现“权限继承”。
